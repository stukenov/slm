# Phase 3 — UI Design Contract

> Visual and interaction contract for the Model Catalog and Model Card pages. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (custom Tailwind components) |
| Icon library | lucide-react ^0.577.0 |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge internal padding |
| sm | 8px | Compact element spacing, pill filter gaps |
| md | 16px | Card internal padding between elements, filter row margin-bottom |
| lg | 24px | Card padding (p-6), section gaps between detail page blocks |
| xl | 32px | Gap between catalog grid and filter row |
| 2xl | 48px (--spacing-content: 3rem) | Content section spacing (between detail page sections) |
| 3xl | 96px (--spacing-section: 6rem) | Page top/bottom padding |

Exceptions: none. Existing `--spacing-section: 6rem` and `--spacing-content: 3rem` tokens are reused from Phase 2.

---

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 | Card descriptions, training details, benchmark values |
| Label | 14px (text-sm) | 500 (medium) | 1.4 | Filter pills, table headers, badge text, field labels |
| Heading | 20px (text-xl) | 600 (semibold) | 1.2 | Section titles on detail page (Architecture, Training, etc.) |
| Display | 28px (text-2xl) | 700 (bold) | 1.2 | Model name on detail page, catalog page title |

Code blocks: JetBrains Mono at 13px, weight 400, line-height 1.6.

Source: Existing landing page uses text-sm (14px) for body, text-lg (18px) for card headings, text-4xl+ for hero. Phase 3 sits between card-level and hero-level typography.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #FFFFFF (--color-background) | Page background, card content areas |
| Secondary (30%) | #F8FAFB (--color-section-bg) | Catalog page background behind card grid, detail page alternating sections, table row striping |
| Accent (10%) | #0EA5E9 (--color-accent-teal) | Filter pill active state, "View on HuggingFace" button, loss chart line, code tab active indicator, "View details" link |
| Destructive | not applicable | No destructive actions in this phase |

Accent reserved for:
1. Active filter pill background (bg-accent-teal, text white)
2. "View on HuggingFace" button background
3. Loss chart line stroke
4. Active code tab underline/indicator
5. Parameter count badge text (accent-teal on accent-teal/10 bg -- matches landing page)
6. "View details" card link text on hover

Supporting colors (from existing tokens):
- #0F172A (--color-slate-900): Headings, model names
- #475569 (--color-slate-600): Body text, descriptions
- #94A3B8 (--color-slate-400): Muted text, inactive pill text, copy button icon
- #E2E8F0 (--color-slate-200): Card borders on white bg, table borders, inactive pill borders
- rgba(255, 255, 255, 0.70) (--color-glass): Glass card background (on gradient sections only)
- rgba(255, 255, 255, 0.30) (--color-glass-border): Glass card border (on gradient sections only)

Glass card note: The frosted glass effect (`bg-glass backdrop-blur-lg border-glass-border`) is only visible against a colored/gradient background. On the catalog page (white/section-bg background), cards use `bg-white border border-slate-200 rounded-xl` instead. The detail page header may use a subtle gradient banner where glass styling applies.

## Component Visual Contracts

### Catalog Page (`/models`)

**Page header:**
- Title: "Models" (Display size, 28px, bold, slate-900)
- Subtitle: 1-line description (Body, 14px, slate-600)
- Spacing: 3xl (96px) top padding, md (16px) between title and subtitle, xl (32px) below subtitle to filters

**Filter pills:**
- Horizontal row, gap-sm (8px) between pills
- Each pill: rounded-full, px-4 py-1.5 (16px horizontal, 6px vertical)
- Inactive: bg-transparent, border border-slate-200, text-slate-600, text-sm font-medium
- Active: bg-accent-teal, text-white, border-transparent, text-sm font-medium
- Hover (inactive): border-slate-400
- Two groups separated by a 16px gap: Type pills | Size pills
- Type group: All, Base, Instruct, GEC
- Size group: All Sizes, < 100M, 100-300M, 300M+
- Margin-bottom: xl (32px) from pills to grid

**Card grid:**
- 3 columns desktop (lg: >= 1024px), 2 columns tablet (md: >= 768px), 1 column mobile
- Gap: lg (24px)
- Each card: bg-white, border border-slate-200, rounded-xl, p-6
- Card content top-to-bottom:
  1. Model display name (text-lg, 18px, font-semibold, slate-900) with HF external link icon (top-right, slate-400, 16px)
  2. Parameter badge (inline-block, bg-accent-teal/10, text-accent-teal, text-sm font-medium, rounded-full, px-3 py-1, mt-2)
  3. Type badge (inline-block, bg-slate-100, text-slate-600, text-xs font-medium, rounded-full, px-2 py-0.5, ml-2 from param badge)
  4. Task label (text-sm, slate-600, mt-3)
  5. "View details" link (text-sm, font-medium, text-accent-teal, mt-4, with right arrow icon 16px)
- Card hover: shadow-md transition-shadow duration-200, border-slate-300
- Card is a clickable link (`<Link>`) wrapping entire card, except HF external link icon

**Empty state (no filter matches):**
- Centered in grid area, py-2xl (48px)
- Icon: Search icon (lucide), 48px, slate-300
- Heading: "No models match your filters" (text-lg, font-semibold, slate-900, mt-md)
- Body: "Try adjusting your filter criteria or view all models." (text-sm, slate-600, mt-sm)
- Action: "Clear filters" text button (text-sm, font-medium, text-accent-teal, mt-md)

### Model Detail Page (`/models/[slug]`)

**Header section:**
- Subtle gradient background (from accent-teal/5 to transparent, 200px height) for visual distinction
- Model display name (Display, 28px, bold, slate-900)
- Full HuggingFace name below (text-sm, font-mono, slate-400, mt-xs)
- Row of badges: param badge + type badge + gated badge (if applicable)
- Gated badge: bg-amber-50, text-amber-700, border border-amber-200, text-xs, rounded-full, px-2 py-0.5, text "Gated - requires approval"
- Description paragraph (Body, 14px, slate-600, mt-md, max-w-2xl)
- "View on HuggingFace" button: bg-accent-teal, text-white, rounded-lg, px-6 py-2.5, text-sm font-medium, with external link icon (16px, ml-2). Hover: bg-accent-teal-dark. Opens new tab.
- Spacing: 3xl top padding, 2xl bottom spacing to first section

**Section pattern (repeats for Architecture, Training, Benchmarks):**
- Section title (Heading, 20px, semibold, slate-900)
- Divider: 1px border-slate-200, mt-sm, mb-md
- Content follows
- Section spacing: 2xl (48px) between sections

**Architecture table:**
- 2-column key-value layout (not a full HTML table)
- Left column: Label (14px, font-medium, slate-600), right: value (14px, regular, slate-900)
- Rows: Architecture Type, Layers, Hidden Size, Attention Heads, Intermediate Size, Vocab Size, Context Length, Tied Embeddings, Activation
- Row spacing: 12px vertical gap
- Background: alternating rows with section-bg (#F8FAFB) and white

**Training details table:**
- Same 2-column key-value layout as Architecture
- Rows: Dataset (with link), Tokens, Hardware, Epochs, Steps, Learning Rate, Batch Size, Precision, Training Time, Optimizer

**Benchmark results table:**
- Full-width table with header row
- Header: bg-section-bg, text-sm font-medium slate-600, px-4 py-2
- Columns: Metric | Value | Description (optional)
- Body rows: text-sm, slate-900, px-4 py-3, border-b border-slate-100
- If no benchmarks: show "Benchmark data coming soon." (text-sm, italic, slate-400)

**Training loss chart:**
- Section title: "Training Loss"
- Chart area: 300px height, full width, rounded-lg border border-slate-200 p-4
- Line color: #0EA5E9 (accent-teal), stroke width 2px, no dots
- Grid: dashed, #E2E8F0
- Axes: text-xs (12px), slate-400
- Tooltip: bg-white, border border-slate-200, rounded-md, shadow-sm, text-sm
- If no loss data: hide entire section (do not show empty chart)

**Code snippet section:**
- Tab bar: two tabs "Python" and "CLI", bottom-bordered
- Active tab: text-accent-teal, border-b-2 border-accent-teal, font-medium
- Inactive tab: text-slate-400, hover text-slate-600
- Tab spacing: gap-lg (24px) between tabs
- Code block: bg-slate-50, border border-slate-200, rounded-lg, p-4, mt-sm
- Code font: JetBrains Mono, 13px, line-height 1.6
- Syntax highlighting theme: github-light (shiki)
- Copy button: absolute top-3 right-3, p-1.5, rounded-md, hover bg-slate-200/50
- Copy icon: 16px, slate-400. After copy: checkmark icon, green-600, reverts after 2s

**Related models:**
- Section title: "Related Models"
- Horizontal row of 2-3 cards (same style as catalog cards but compact)
- Grid: flex, gap-md (16px), overflow-x-auto on mobile
- Each card: min-w-[260px], same styling as catalog card

**Breadcrumb (top of detail page):**
- "Models" (text-sm, text-accent-teal, link) > "Model Name" (text-sm, slate-600)
- Separator: chevron-right icon, 14px, slate-400
- mb-lg (24px) below breadcrumb

## Interaction Contracts

| Interaction | Behavior |
|-------------|----------|
| Filter pill click | Updates URL query params (`?type=base&size=large`). Grid re-renders instantly (client-side). "All" removes param. |
| Card click (catalog) | Navigates to `/models/[slug]` via Next.js Link. |
| HF icon click (catalog card) | Opens HuggingFace URL in new tab. Stops event propagation (does not navigate to detail page). |
| "View on HuggingFace" button (detail) | Opens HuggingFace URL in new tab. |
| Code tab switch | Switches visible code block. No URL change. Client-side state only. |
| Copy button click | Copies raw code to clipboard. Icon changes to checkmark for 2 seconds, then reverts. |
| Loss chart hover | Tooltip shows step number and loss value at cursor position. |
| Related model card click | Navigates to that model's detail page. |
| Breadcrumb "Models" click | Navigates back to `/models` catalog. |
| "Clear filters" (empty state) | Removes all URL query params, shows all models. |
| Browser back/forward | Filter state restores from URL params (source of truth). |
