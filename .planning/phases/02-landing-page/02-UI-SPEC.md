## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` (white) | Page background, section backgrounds |
| Secondary (30%) | `rgba(255,255,255,0.15)` over gradient / `#F8FAFB` on flat sections | Frosted glass cards, model cards, impact section background |
| Accent (10%) | `#0EA5E9` (sky-500, teal/cyan) | Primary CTA buttons, gradient anchor color, "View all" links, badge highlights |
| Accent gradient end | `#06B6D4` (cyan-500) | Gradient secondary stop — hero background animates between accent and this value |
| Destructive | not applicable | No destructive actions in this phase |

Accent reserved for:
- Primary CTA button backgrounds ("Try Playground", "Browse Models", bottom CTA)
- Hero animated gradient background (accent to cyan-500, with white/light teal tints)
- "View all 7 models" link text color
- Floating model size badge borders/highlights
- HuggingFace icon link hover state on model cards

**Hero gradient definition:** Linear gradient from `#0EA5E9` to `#06B6D4` at 135deg, with a subtle white radial overlay. Animation: `background-position` shift over 8s, `ease-in-out`, infinite loop.

**Frosted glass cards:** `background: rgba(255, 255, 255, 0.70)`, `backdrop-filter: blur(12px)`, `border: 1px solid rgba(255, 255, 255, 0.30)`, `border-radius: 12px`.

**Text colors:**
- Primary text: `#0F172A` (slate-900)
- Secondary text: `#475569` (slate-600)
- Text on gradient (hero): `#FFFFFF` with `text-shadow: 0 1px 2px rgba(0,0,0,0.1)`

## Component Inventory

### 1. Hero Section
- Full-width gradient background (animated, 135deg, accent colors)
- Display heading (48px/32px mobile) + subtitle body text (16px)
- Floating model size badges: `50M`, `150M`, `300M`, `600M` — pill-shaped, frosted glass style, absolute-positioned with subtle float animation (translateY +/- 6px, 3s ease-in-out, staggered)
- Mobile: badges replaced with inline text row `50M · 150M · 300M · 600M` at label size (14px)
- Two CTA buttons side by side (primary + secondary/outline)
- Breakpoint: badges visible at `md` (768px)+, inline text below `md`

### 2. Model Highlight Cards (x3)
- Section heading (20px semibold) + subtitle (16px regular)
- 3-column grid (desktop), horizontal swipe carousel (mobile < 768px)
- Each card: frosted glass style, contains:
  - Model name (heading weight)
  - Parameter count badge (label size)
  - Architecture name (body text, secondary color)
  - Training data summary (body text, secondary color)
  - HuggingFace icon link (secondary action, top-right corner)
  - Full card is clickable, links to `/models/{slug}`
- "View all 7 models" link below cards (accent color, arrow suffix)
- Mobile carousel: horizontal scroll-snap, 85vw card width, 16px gap

### 3. Impact / Social Proof Section
- Light grey background (`#F8FAFB`)
- Section heading + subtitle
- Tilqazyna case study card: larger card with quote-style layout, Tilqazyna name, description of GEC integration
- Stats row: 3 stat blocks inline — "26 experiments" / "7 published models" / "9B tokens trained"
- Stat number: heading size (20px), semibold; stat label: label size (14px), secondary color

### 4. Docs/SDK Teaser Section
- White background
- Section heading + 2-3 lines of body text describing developer tools
- Single text link to `/docs` (accent color, arrow suffix)
- No code snippet per CONTEXT.md decision

### 5. Bottom CTA Banner
- Full-width, accent gradient background (same as hero)
- Single line heading (white text): "Ready to build with Kazakh NLP?"
- Single primary CTA button (white background, accent text — inverted from hero CTAs)

### 6. CTA Button Variants

| Variant | Background | Text | Border | State |
|---------|-----------|------|--------|-------|

## Scroll Animations

All sections use scroll-triggered fade-in:
- **Trigger:** Element enters viewport at 80% scroll position
- **Animation:** `opacity: 0 -> 1` + `translateY: 20px -> 0`
- **Duration:** 600ms
- **Easing:** `cubic-bezier(0.16, 1, 0.3, 1)` (ease-out expo)
- **Stagger:** Cards within a section stagger by 100ms each
- **Implementation:** CSS `@keyframes` + Intersection Observer (no heavy animation library)

## Accessibility

- All CTA buttons have visible focus rings (`outline: 2px solid #0EA5E9`, `outline-offset: 2px`)
- Disabled buttons use `aria-disabled="true"` (not `disabled` attribute) so screen readers announce them with their label + "Coming soon"
- Hero gradient text must maintain WCAG AA contrast (white on `#0EA5E9` = 3.3:1 — use `text-shadow` or ensure gradient darkens to at least `#0284C7` where text overlays)
- Floating badges have `aria-hidden="true"` (decorative)
- Model cards are `<a>` elements with descriptive `aria-label` including model name
- Scroll animations respect `prefers-reduced-motion: reduce` — disable all animations, show content immediately
- Language: `<html lang>` attribute set per locale by next-intl

---

## Performance Contract

- No external API calls at runtime
- No client-side data fetching
- Model data: hardcoded JSON, statically rendered at build time
- Hero gradient animation: CSS-only (no JS animation loop)
- Floating badges: CSS `@keyframes` only
- Intersection Observer for scroll animations: single observer instance, disconnect after all sections revealed
- Images: none required for landing page (text + gradient + CSS effects only)
- Target: Lighthouse Performance score >= 95
