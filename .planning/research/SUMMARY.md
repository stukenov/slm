# Research Summary

## Stack
Next.js 15+ (App Router) + Tailwind 4 + next-intl for i18n. Content as MDX/JSON in git (no database). Hosted on Vercel, Cloudflare DNS. Blog posts as MDX files generated by AI script, committed to repo. API proxy via Edge routes to kaznu:15127.

## Table Stakes
- Marketing landing page (Linear/Stripe minimalism)
- Model catalog with cards, metrics, HF links, code snippets
- Playground (GEC + text generation, proxied through Vercel)
- Documentation (quickstart, API reference, fine-tuning guide)

## Differentiators
- **Kazakh NLP Leaderboard** — no equivalent exists, positions platform as authority
- **People/Company ratings** — maps the ecosystem, creates network effects
- **SEO content engine** — 10 posts/day in 3 languages for organic traffic
- **Journey/Whitepaper** — unique narrative from 14M to 600M
- **Success stories** — Tilqazyna GEC integration proof

## Watch Out For
1. **SEO spam detection** — start at 2-3 posts/day, include real data, monitor indexing
2. **kaznu server SPOF** — graceful degradation, health checks, static fallbacks
3. **Trilingual quality** — use own GEC model to proofread Kazakh content
4. **Leaderboard bias** — publish methodology, include all models, multiple benchmarks
5. **Overbuilding** — launch landing + catalog first, validate traffic before SDK/API

## Build Order Recommendation
1. Project setup + landing page
2. Model catalog + cards
3. Playground (needs API proxy)
4. Blog engine + initial content
5. Documentation
6. Leaderboard + ratings
7. SEO content pipeline (automation)
8. SDK + hosted API
9. Journey/success stories pages
