# exp_003 — Арифметический пайплайн KK→EN→Think→KK

## Задача
Три модели в цепочке: перевод KK→EN, "думание" на английском (решение арифметики), перевод EN→KK.

## Контекст
Пользователь предложил архитектуру "билингвального мозга" — модель принимает вопрос на казахском, переводит на английский, решает задачу на английском, переводит ответ обратно. Выбран домен арифметики (слова, числа 0-10, +/-) как минимальный proof-of-concept.

## Архитектура

| Модель | Тип | Параметры | Задача |
|--------|-----|-----------|--------|
| M1 (KK→EN) | Encoder-Decoder Transformer | 240K | Перевод вопроса |
| M2 (Thinker) | Decoder-only Transformer | 105K | Решение арифметики |
| M3 (EN→KK) | Encoder-Decoder Transformer | 240K | Перевод ответа |

Все: dim=64, 4 heads, 2 layers, word-level.

## Данные
- 132 примера: числа 0-10, операции (қосу/plus, алу/minus)
- Словари: KK=21 слов, EN=20 слов
- Формат: `"екі қосу үш нешеге тең ?" → "what is two plus three ?" → "two plus three is five ." → "екі қосу үш — бес ."`

## Результаты

### Обучение (200 шагов, Metal M1 GPU)

| Модель | Loss start | Loss end | Время | Speed |
|--------|-----------|----------|-------|-------|
| M1 KK→EN | 3.39 | 0.002 | 75s | 2.65 s/s |
| M2 Thinker | 3.37 | 0.433 | 33s | 6.05 s/s |
| M3 EN→KK | 3.15 | 0.002 | 63s | 3.16 s/s |

### Инференс: 4/6 (67%)
- M1/M3 переводят идеально
- M2 ошибается: `10-5=7` и `6+3=7` (недообучен, loss=0.43)

## Выводы
- Encoder-decoder сходится быстро на переводе (почти 1-к-1 маппинг слов)
- Decoder-only для арифметики нужно больше шагов — задача требует "вычисления", а не маппинга
- Пайплайн end-to-end работает: KK вопрос → KK правильный ответ (когда M2 не ошибается)

## К чему хотим прийти
- Больше шагов / больше батч для M2
- Больший диапазон чисел
- Более сложные операции (умножение)
