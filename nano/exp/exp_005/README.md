# exp_005 — Мультиязычный пайплайн с роутером (KK+RU→EN→Think→KK/RU)

## Задача
4 модели в цепочке: Router определяет язык, M1 переводит на EN, M2 думает, M3 переводит обратно на язык оригинала.

## Контекст
exp_004 показал что decoder-only работает лучше encoder-decoder. Теперь добавляем второй язык (русский) и роутер. Мультиязычные M1/M3 вместо отдельных моделей на каждый язык.

## Архитектура

| Модель | Параметры | Задача |
|--------|-----------|--------|
| Router | 3,906 | Классификатор языка (kk/ru) |
| M1 (Multi→EN) | 109,497 | Перевод KK/RU→EN с тегом языка |
| M2 (Thinker) | 109,497 | Арифметика на EN |
| M3 (EN→Multi) | 109,497 | Перевод EN→KK/RU с тегом языка |

Все decoder-only, единый словарь 57 слов, dim=64, 4 heads, 2 layers.

## Данные
- 264 примера (132 KK + 132 RU), числа 0-10, +/-
- KK: "бір қосу екі нешеге тең ?" / RU: "сколько будет два плюс три ?"

## Результаты

### Обучение (300 шагов, Metal M1)
| Модель | Loss start → end | Время |
|--------|-----------------|-------|
| Router | 0.668 → 0.000 | 5s |
| M1 | 4.204 → 0.001 | 57s |
| M2 | 4.273 → 0.001 | 52s |
| M3 | 4.356 → 0.088 | 51s |

### Инференс: 8/8 (100%), Router: 8/8 (100%)

## Выводы
- Router тривиален (KK и RU лексика не пересекается)
- M2 сошёлся до 0.001 (vs 0.036 в exp_004) — удвоение EN данных помогло
- Мультиязычные M1/M3 работают — язык управляется тегом `<kk>`/`<ru>`
- M3 loss чуть выше (0.088) — сложнее генерить на двух языках
