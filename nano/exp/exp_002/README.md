# exp_002 — Character-level Transformer на tinygrad

## Задача
Создать маленький языковой модель на символах (Unicode) с обучением и генерацией.

## Контекст
После exp_001 (XOR) пользователь попросил "то же самое но с маленьким языковым моделем на символах". Выбран казахский текст как релевантный проекту SLM.

## Что привело к этому
exp_001 показал что tinygrad работает. Следующий шаг — проверить на реальной задаче NLP: авторегрессионная генерация текста.

## Архитектура
- GPT-style decoder-only Transformer
- 2 слоя, 4 головы внимания, dim=64
- Символьный уровень (53 уникальных символа из корпуса)
- Контекст: 64 символа
- 111K параметров
- Loss: Cross-Entropy, Optimizer: Adam (lr=3e-3)

## Данные
- 729 символов казахского текста (встроенный в код)
- Без токенизатора — каждый Unicode-символ = токен

## Результаты
- 600 шагов, loss: 4.03 → 0.07, perplexity: 56 → 1.1
- Модель запомнила корпус наизусть (ожидаемо: 111K params >> 729 символов)
- Генерация по промптам "Қазақ", "Астана", "Мәдени" — воспроизводит текст почти дословно
- На границе контекста начинает галлюцинировать

## К чему хотим прийти
- Масштабирование: больший корпус, subword tokenizer, больше слоёв
- Проверить обобщение (а не запоминание) при увеличении данных

## Запуск
```bash
cd nano && .venv/bin/python exp/exp_002/char_lm.py
```
