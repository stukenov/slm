# exp_006 — Think-оркестратор с генерацией плана действий

## Что просили
Входящие запросы на трёх языках (KK, RU, EN). Think — decoder-only модель, которая сама генерирует план действий (action-токены). Если вход на английском — перевод не нужен, напрямую в Math. Если на другом языке — Think решает перевести, посчитать, перевести обратно.

## Что привело к этому
- exp_003: 3 модели (enc-dec), 67% точность
- exp_004: 3 модели (all decoder-only), 88%
- exp_005: 4 модели + classifier Router, KK+RU, 100%
- exp_006: заменяем Router на Think (decoder-only), добавляем EN, Think сам генерирует план

## Архитектура

4 модели (все decoder-only, dim=64, 4 heads, 2 layers, ~110K params каждая):

| Модель | Формат | Задача |
|--------|--------|--------|
| Think | `<think> [input] <plan> [actions] <eos>` | Генерирует план действий |
| M1 | `<kk>/<ru> [src] <en> [en] <eos>` | Мультиязычный переводчик →EN |
| Math | `<q> [en_q] <a> [en_a] <eos>` | Арифметический решатель |
| M3 | `<en> [en_a] <kk>/<ru> [src_a] <eos>` | Мультиязычный переводчик ←EN |

### Action-токены
- `<lang_kk>`, `<lang_ru>`, `<lang_en>` — определённый язык
- `<translate>` — нужен перевод на EN (вызвать M1)
- `<math>` — отправить в Math
- `<translate_back>` — перевести ответ обратно (вызвать M3)

### Два шаблона плана
- Не-EN: `<lang_X> <translate> <math> <translate_back>`
- EN: `<lang_en> <math>`

## Данные
396 примеров (132 KK + 132 RU + 132 EN), числа 0-10, +/-
